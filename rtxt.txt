2. Как показать сайт заказчику
Ты абсолютно правильно использовал serveo.net и localtunnel. Это отличные инструменты для временной демонстрации. Принцип работы с Django точно такой же.

Твоя команда python manage.py runserver по умолчанию запускает сервер на порту 8000. Значит, тебе нужно "пробросить" именно этот порт.

Алгоритм действий:

Настрой ALLOWED_HOSTS. Это самое важное! В файле photographer_project/settings.py (или как называется твой файл с настройками), найди ALLOWED_HOSTS и добавь туда '.localhost' и '*'.
Внимание: '*' — это небезопасно для боевого сервера, но для быстрой демонстрации — это самый простой способ.

Python

# photographer_project/settings.py

ALLOWED_HOSTS = ['127.0.0.1', '.localhost', '*'] 

Запусти сервер Django. В одном терминале выполни:
python manage.py runserver 0.0.0.0:8000
(Использование 0.0.0.0 вместо 127.0.0.1 делает сервер доступным по всем сетевым интерфейсам твоего компьютера, что более надежно для таких туннелей).

Запусти туннель. В другом терминале выполни команду localtunnel, указав порт 8000:
npx localtunnel --port 8000

После этого localtunnel выдаст тебе публичный URL вида https://some-random-name.loca.lt.  Эту ссылку ты можешь отправить заказчику, и он увидит твой локально запущенный сайт. Он будет работать, пока открыты оба твоих терминала.

Надеюсь, эти решения помогут тебе продвинуться еще дальше в разработке нашего сайта!
